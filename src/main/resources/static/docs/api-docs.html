공지 API

HTTP 요청

요청 메세지

GET /api/v1/notice?type=bch&offset=0&max=10 HTTP/1.1
Accept: application/json
Host: kuring.herokuapp.com

요청 파라미터

Parameter Description Constraints

type

공지 카테고리 키워드

bch, sch, emp, nat, stu, ind, nor, lib

offset

가져올 공지의 시작 인덱스

0 이상의 정수

max

가져올 공지 최대 개수

1 이상 30 이하의 정수


HTTP 응답

응답 메세지

HTTP/1.1 200 OK
Content-Type: application/json
Content-Length: 298

{
  "isSuccess" : true,
  "resultMsg" : "성공",
  "resultCode" : 200,
  "baseUrl" : "${notice.normal-base-url}",
  "noticeList" : [ {
    "articleId" : "5cw2e1",
    "postedDate" : "20211016",
    "subject" : "[학사] 2021년 학사 관련 공지",
    "category" : "bachelor"
  } ]
}

응답 필드

Path Type Description

isSuccess

Boolean

성공 여부

resultMsg

String

결과 메세지

resultCode

Number

결과 코드

baseUrl

String

공지 확인할 수 있는 url의 공통 부분

noticeList[].articleId

String

공지 ID

noticeList[].postedDate

String

공지 게시일

noticeList[].subject

String

공지 제목

noticeList[].category

String

공지 카테고리명


잘못된 HTTP 요청/응답 예시

1. 잘못된 카테고리 요청

GET /api/v1/notice?type=invalid-type&offset=0&max=20 HTTP/1.1
Accept: application/json
Host: kuring.herokuapp.com
{
  "isSuccess" : false,
  "resultMsg" : "해당 공지 카테고리를 지원하지 않습니다.",
  "resultCode" : 400
}

2. 잘못된 offset 혹은 max

GET /api/v1/notice?type=bch&offset=-1&max=20 HTTP/1.1
Accept: application/json
Host: kuring.herokuapp.com
{
  "isSuccess" : false,
  "resultMsg" : "파라미터 값 중 잘못된 값이 있습니다.",
  "resultCode" : 400
}

  "resultCode" : 401
}
2. 요청 JSON body 필드 누락

Content-Length: 48
Host: kuring.herokuapp.com

{
  "categories" : [ "bachelor", "student" ]
}
{
  "isSuccess" : false,
  "resultMsg" : "필수 파라미터가 없습니다.",
  "resultCode" : 400
}
3. 지원하지 않는 카테고리

Content-Type: application/json

Content-Length: 81
Host: kuring.herokuapp.com

{
  "id" : "TEST_TOKEN",
  "categories" : [ "bachelor", "invalid-category" ]
}
{
  "isSuccess" : false,
  "resultMsg" : "파라미터 값 중 잘못된 값이 있습니다.",
  "resultCode" : 400
